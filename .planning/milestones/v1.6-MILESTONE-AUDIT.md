# v1.6 Milestone Audit: Repo Memory & Experience Cards

**Completed:** 2026-05-16
**Archived:** 2026-05-17

## Summary

v1.6 delivered the repo memory layer from the original product vision: durable, source-linked, source-free memory cards that can be reviewed and selected into plans and packs without turning memory into an always-injected context dump.

## Delivered

- Source-free `MemoryCard` and `SelectedMemory` contracts with stable camelCase JSON.
- SQLite `memory_cards` persistence, migration metadata, review state, disable/reject controls, and storage diagnostics.
- Deterministic domain memory cards derived from safe inventory, symbols, tests, docs, and dependency edges.
- Source-free experience cards generated from local eval traces and structured metadata rather than raw prompts, transcripts, terminal logs, or source snippets.
- Bounded memory selection in `prepare_task`, `get_pack`, MCP tool responses, and pack resources.
- Memory review commands, docs, release docs, and deterministic memory smoke coverage.

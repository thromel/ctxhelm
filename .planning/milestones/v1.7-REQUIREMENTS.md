# Requirements: Repo Context Packer v1.7 Adaptive Retrieval Policy & Feedback Loop

**Defined:** 2026-05-17
**Milestone:** v1.7 Adaptive Retrieval Policy & Feedback Loop
**Core Value:** Given a coding task, ctxpack should return the smallest safe evidence set that makes an existing coding agent more likely to inspect the right files, run the right tests, and avoid irrelevant context.

## v1.7 Requirements

Requirements for learning from source-free session feedback, reporting policy quality, tuning retrieval weights safely, and proving whether pack depth improves agent outcomes.

### Feedback Events

- [x] **LEARN-01**: Maintainer can record source-free session feedback events for recommended, read, edited, tested, passed, failed, blocked, and user-corrected files.
- [x] **LEARN-02**: Feedback ingestion accepts only path lists, command labels, outcome enums, task hashes, pack IDs, budgets, target agents, and source-free notes/tags; it rejects raw prompts, terminal logs, source snippets, secrets, and cloud payloads.
- [x] **LEARN-03**: Feedback events are stored locally with deterministic IDs, timestamps, privacy status, schema versioning, and compatibility diagnostics.
- [x] **LEARN-04**: CLI and MCP-compatible JSON output can list recent feedback events and summarize ingestion status without adding a large new MCP tool surface.

### Policy Reporting

- [x] **LEARN-05**: Maintainer can generate source-free policy reports that compare recommended files/tests with read, edited, tested, passed, failed, and corrected files.
- [x] **LEARN-06**: Reports include context precision, read precision, edit recall proxy, validation coverage, repeated missing-file families, signal contribution, and token ROI where pack metadata is available.
- [x] **LEARN-07**: Reports group evidence by task type, target agent, pack budget, retrieval signal, and repository area without storing source text or prompts.
- [x] **LEARN-08**: Reports surface degraded or low-sample diagnostics instead of pretending weak feedback evidence is conclusive.

### Adaptive Policy Tuning

- [x] **LEARN-09**: Maintainer can derive candidate retrieval-policy weight changes from benchmark and feedback evidence without mutating the active policy by default.
- [x] **LEARN-10**: Candidate policies include source-free rationale, before/after metrics, regression warnings, sample counts, and rollback metadata.
- [x] **LEARN-11**: Tuning protects exact path, symbol, test, privacy, and active-anchor signals from being demoted below safe minimums.
- [x] **LEARN-12**: Maintainer can apply, inspect, disable, and roll back local policy profiles with JSON and Markdown output.

### Agent Outcome Comparison

- [x] **LEARN-13**: Maintainer can compare plan-only, brief, standard, and deep pack outcomes across fixed tasks using source-free feedback and eval metadata.
- [x] **LEARN-14**: Outcome reports include pass rate, blocked rate, correction rate, validation-command coverage, average recommended context size, and useful target files per 1k estimated tokens.
- [x] **LEARN-15**: Comparison tooling distinguishes retrieval-quality improvements from changed samples, low-information tasks, and missing validation evidence.
- [x] **LEARN-16**: Product proof can include v1.7 policy/feedback summary sections while preserving local-only source-free guarantees.

### Documentation and Release Gates

- [x] **LEARN-17**: Documentation explains feedback events, policy reports, tuning profiles, rollback, privacy guarantees, and when feedback should not affect retrieval.
- [x] **LEARN-18**: Release gate includes deterministic feedback and policy smoke coverage proving source-free ingestion, report generation, candidate tuning, rollback metadata, and outcome comparison.
- [x] **LEARN-19**: RefactoringMiner or another large-history repo remains the external smoke target when practical, with fixed sample boundaries before claiming improvement.
- [x] **LEARN-20**: Existing `prepare_task`, `get_pack`, MCP, memory, semantic, precision, and storage behavior remains backward-compatible unless an explicit profile is enabled.

## Out of Scope

Explicitly excluded from v1.7 to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Autonomous code editing | ctxpack remains a read-only context broker; agents own edits and permissions. |
| Cloud feedback upload | v1.7 feedback remains local and source-free by default. |
| Raw transcript learning | Prompt text, terminal logs, and raw model output are too privacy-sensitive for default storage. |
| Hosted team policy sync | v2.0 owns workspace/team sharing. |
| UI pack inspector | v2.1 owns diagnostics UI. |
| Unbounded online learning | v1.7 proposes, applies, and rolls back explicit local profiles; it does not silently mutate retrieval behavior. |
| Replacing lexical/search baselines | v1.7 tunes signals but preserves exact-path, symbol, test, and anchor safety floors. |

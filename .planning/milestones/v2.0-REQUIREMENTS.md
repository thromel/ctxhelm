# Requirements: Repo Context Packer v2.0 Workspace & Team Layer

**Defined:** 2026-05-17
**Milestone:** v2.0 Workspace & Team Layer
**Core Value:** Given a coding task, ctxpack should return the smallest safe evidence set that makes an existing coding agent more likely to inspect the right files, run the right tests, and avoid irrelevant context.

## v2.0 Requirements

Requirements for multi-repo workspaces, source-free shared artifacts, team privacy policy templates, and agent-native workspace use.

### Workspace Inventory

- [ ] **WORK-01**: Maintainer can define a local workspace manifest with multiple repository roots, stable repo IDs, display labels, and optional package/service tags.
- [ ] **WORK-02**: Maintainer can generate a source-free workspace inventory report that summarizes repo freshness, safe inventory counts, storage status, cards, evals, and privacy status without source snippets.
- [ ] **WORK-03**: Workspace inventory detects missing, moved, duplicated, ignored, generated, sensitive, stale, and inaccessible repositories with actionable diagnostics.
- [ ] **WORK-04**: Workspace state remains local by default and stores only repo labels, paths, hashes, counts, timestamps, policy IDs, and source-free artifact references.

### Workspace Context Planning

- [ ] **PLAN-01**: Agent or maintainer can run workspace-aware `prepare-task` for a task and receive repo routing recommendations before file-level retrieval.
- [ ] **PLAN-02**: Workspace context plans preserve repository boundaries with repo ID, repo label, path labels, line hints, evidence reasons, and confidence per target.
- [ ] **PLAN-03**: Workspace planning runs per-repo retrieval only inside selected repositories unless explicit evidence justifies wider expansion.
- [ ] **PLAN-04**: Workspace packs include source-free workspace provenance, selected repo rationale, validation guidance, and budget warnings while preserving existing single-repo behavior by default.

### Shared Source-Free Artifacts

- [ ] **SHARE-01**: Maintainer can export source-free shared artifact manifests for context cards, benchmark reports, policy profiles, feedback summaries, and proof reports.
- [ ] **SHARE-02**: Maintainer can import and inspect shared artifact manifests without importing raw source, prompts, terminal logs, secrets, or model transcripts.
- [ ] **SHARE-03**: Shared artifacts include freshness, derivation, privacy status, source-free guarantees, and compatibility diagnostics.
- [ ] **SHARE-04**: Shared artifacts can be disabled, regenerated, or ignored locally without mutating source repositories or agent editing behavior.

### Team Privacy Policies

- [ ] **PRIV-01**: Maintainer can create source-free team privacy policy templates for workspace indexing, artifact export, optional cloud features, and redaction defaults.
- [ ] **PRIV-02**: Workspace and artifact commands enforce deny-by-default handling for secrets, source snippets, prompts, terminal logs, local dumps, credentials, and generated outputs.
- [ ] **PRIV-03**: Policy reports identify allowed, blocked, degraded, and redacted workspace artifacts with clear reasons and no source leakage.
- [ ] **PRIV-04**: Local policy profiles remain explicit, inspectable, reversible, and compatible with v1.7 feedback/profile controls.

### Agent-Native Workspace Integration

- [ ] **AGENT-01**: CLI JSON and Markdown outputs expose workspace-aware planning, pack, status, artifact, and policy results with stable typed contracts.
- [ ] **AGENT-02**: MCP resources expose workspace summaries and shared artifact manifests without growing the MCP tool surface unnecessarily.
- [ ] **AGENT-03**: `AGENTS.md`, Cursor, Claude Code, Codex, and OpenCode guidance explain when agents should call workspace-aware ctxpack flows.
- [ ] **AGENT-04**: Release smokes prove single-repo compatibility, workspace manifest behavior, source-free artifact export/import, team policy enforcement, and workspace MCP/resource output.

## Out of Scope

Explicitly excluded from v2.0 to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Hosted source indexing | v2.0 is local-first workspace support, not a SaaS source index. |
| Raw source artifact sharing | Team sharing must remain source-free by default. |
| Autonomous code editing | ctxpack remains a read-only context broker; agents own edits and permissions. |
| UI pack inspector | v2.1 owns diagnostics UI. |
| Cloud embeddings/reranking by default | Optional cloud features require explicit policy controls and remain outside v2.0 defaults. |
| Enterprise admin/SSO | Useful later, but v2.0 focuses on local team-safe contracts and artifacts. |
| Unbounded cross-repo retrieval | Workspace planning must route first and retrieve second under budget caps. |

# Roadmap: ctxhelm

## Overview

This roadmap opens v2.0 Workspace & Team Layer. v1.3 made local storage durable, v1.6 added source-free repo memory, and v1.7 added source-free feedback and policy reporting. v2.0 extends those foundations from one repository to local multi-repo workspaces and team-safe source-free artifacts.

## Phase Details

### Phase 35: Workspace Manifest & Inventory

**Goal**: Maintainers can define a multi-repo workspace and inspect source-free workspace status, freshness, privacy, and diagnostics.

**Depends on**: v1.3 storage, v1.6 cards/memory, v1.7 feedback policy reports

**Requirements**: WORK-01, WORK-02, WORK-03, WORK-04

**Success Criteria** (what must be TRUE):
1. Workspace manifest contracts support multiple local repo roots with stable repo IDs, labels, and tags.
2. Workspace status reports aggregate per-repo safe inventory, storage, cards, eval, feedback, and privacy metadata without source snippets.
3. Missing, moved, duplicated, ignored, generated, sensitive, stale, and inaccessible repo states produce actionable diagnostics.
4. Existing single-repo commands remain backward-compatible when no workspace manifest is provided.

**Plans**: 1 plan

Plans:
- [x] 35-workspace-manifest-inventory-01-PLAN.md — Add workspace contracts, manifest parsing, source-free status aggregation, diagnostics, and tests.

### Phase 36: Workspace-Aware Context Planning

**Goal**: ctxhelm routes tasks to likely repositories before retrieval and emits repo-boundary-aware plans and packs.

**Depends on**: Phase 35

**Requirements**: PLAN-01, PLAN-02, PLAN-03, PLAN-04

**Success Criteria** (what must be TRUE):
1. Workspace-aware prepare-task recommends likely repositories before file-level retrieval.
2. Workspace plans and packs preserve repo IDs, labels, path labels, line hints, evidence reasons, confidence, and workspace provenance.
3. Per-repo retrieval runs only inside selected repositories unless explicit evidence justifies wider workspace expansion.
4. Budget warnings and validation guidance remain source-free and single-repo behavior remains unchanged by default.

**Plans**: 1 plan

Plans:
- [x] 36-workspace-aware-context-planning-01-PLAN.md — Add task-to-repo routing, workspace plan/pack output, budget controls, and compatibility tests.

### Phase 37: Shared Artifacts & Team Policies

**Goal**: Maintainers can export/import source-free shared artifact manifests and enforce team privacy policy templates.

**Depends on**: Phase 36

**Requirements**: SHARE-01, SHARE-02, SHARE-03, SHARE-04, PRIV-01, PRIV-02, PRIV-03, PRIV-04

**Success Criteria** (what must be TRUE):
1. Shared artifact manifests cover context cards, benchmark reports, policy profiles, feedback summaries, and proof reports without raw source, prompts, logs, secrets, or transcripts.
2. Import and inspect flows validate source-free guarantees, compatibility, freshness, derivation, privacy status, and disabled/regenerated states.
3. Team privacy policy templates control workspace indexing, artifact export, optional cloud-feature allowances, and redaction defaults.
4. Policy reports explain allowed, blocked, degraded, and redacted artifacts with reversible local state.

**Plans**: 1 plan

Plans:
- [x] 37-shared-artifacts-team-policies-01-PLAN.md — Add source-free artifact manifests, import/export/inspect flows, team policy templates, enforcement, and tests.

### Phase 38: Agent-Native Workspace Release Gates

**Goal**: CLI, MCP resources, adapter guidance, docs, and release smokes prove workspace behavior without breaking single-repo flows.

**Depends on**: Phase 37

**Requirements**: AGENT-01, AGENT-02, AGENT-03, AGENT-04

**Success Criteria** (what must be TRUE):
1. CLI JSON and Markdown outputs expose stable workspace planning, pack, status, artifact, and policy contracts.
2. MCP resources expose workspace summaries and shared artifact manifests without adding a large tool surface.
3. AGENTS.md, Codex, Claude Code, Cursor, and OpenCode guidance explain workspace-aware ctxhelm usage and fallback behavior.
4. Release smokes prove single-repo compatibility, workspace manifest behavior, artifact export/import, policy enforcement, and source-free workspace resources.

**Plans**: 1 plan

Plans:
- [x] 38-agent-native-workspace-release-gates-01-PLAN.md — Add docs, adapter guidance, MCP resource proof, smoke scripts, and release-gate coverage.

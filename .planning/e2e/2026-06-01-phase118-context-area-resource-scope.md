# Phase 118 E2E: Context Area Resource Scope

Date: 2026-06-01

## Objective

Make dynamic context-area MCP resource semantics explicit. Agents should be able
to distinguish inventory-wide resource counts from task-conditioned plan
`roleCounts` and `selectedRoleCounts`.

## Commands

```bash
CARGO_NET_OFFLINE=true CARGO_TARGET_DIR=/tmp/ctxpack-phase118-target \
  CTXPACK_SMOKE_REPO="$(pwd)" \
  CTXPACK_SMOKE_PATH="crates/ctxpack-mcp/src/resources.rs" \
  CTXPACK_SMOKE_TASK="verify context area resource scope metadata" \
  bash scripts/smoke-mcp-protocol.sh
```

The source-free proof artifact was generated by launching `ctxpack serve-mcp`
from a temporary non-repo cwd, calling `prepare_task` with an explicit repo, and
reading:

- `ctxpack://repo/context-areas`
- `ctxpack://repo/context-area/crates%2Fctxpack-mcp`

## Result

Durable proof:

- `.ctxpack/e2e/phase118-context-area-resource-scope.json`

The proof records:

- `serverCwdInsideRepo = false`
- `explicitRepoPrepareTask = true`
- `contextAreasResourceScope.kind = safeInventoryArea`
- `contextAreasResourceScope.taskConditioned = false`
- `contextAreasResourceScope.countsSource = safeInventory`
- `contextAreasResourceScope.pathSource = safeInventory`
- the same `resourceScope` fields on the specific context-area resource

## Boundary

- This is additive MCP resource metadata.
- It does not change the MCP tool surface.
- It does not alter plan-level target-file ranking.
- It does not store source snippets or raw MCP traffic.
